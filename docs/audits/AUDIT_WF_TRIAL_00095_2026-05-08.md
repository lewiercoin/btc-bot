## Safety Flag Evaluation

### pf_hard_review_required: FALSE POSITIVE (non-blocking)

**Trigger condition:** Window 2 val PF = 7.5001 (> 6.0 threshold)

**Evidence for false positive:**
1. Window 1 val PF = 4.84 (< 6.0, no flag)
2. Full-range PF = 4.66 (reasonable)
3. Window 2 has only 33 trades (small sample variance expected)
4. Window 2 val Sharpe = 12.21 (consistent with window 1 Sharpe = 14.27, not artifact-level divergence)

**Evidence for true positive:**
1. PF = 7.50 is elevated and could indicate "a few large winners" luck
2. Window 2 DD = 1.94% is very low (lucky streak signal)

**Verdict:** **NON-BLOCKING FALSE POSITIVE**

**Rationale:** Window 1's 106-trade sample with PF=4.84 is the stronger evidence. Window 2's PF=7.50 is likely sample variance from small N=33. If this were a systematic artifact (e.g., parameter overfitting, data leakage), we'd expect both windows to show extreme PF. Instead, window 1 shows moderate PF, suggesting window 2 is noise, not signal.

**Monitoring condition:** Track live PF. If live PF remains >6.0 after 50+ trades, re-evaluate for artifact.

### oos_outperformance_review_required: FALSE POSITIVE (non-blocking)

**Trigger condition:** OOS performed 43-45% better than train in both windows (negative degradation)

**Evidence for false positive:**
1. **Consistency:** Both windows show OOS outperformance (not random luck in one window)
2. **Window 1 credibility:** 106-trade sample with ER=2.46 is credible evidence
3. **Architectural validation:** trial-00095 follows gate vs premium pattern from V3 audit:
   - weight_sweep_detected = 0.150 (LOW) → sweep is gate, not premium
   - weight_reclaim_confirmed = 3.750 (HIGH) → quality filter edge
   - weight_tfi_impulse = 4.900 (HIGH) → momentum edge
   - weight_ema_trend_alignment = 5.000 (HIGH) → trend edge
4. **Regime hypothesis:** 2024-2026 OOS period might favor this param combination (quality filters + trend alignment) more than 2022-2024 train period
5. **Full-range metrics are not extreme:** ER=2.13, PF=4.66 are strong but credible

**Evidence for true positive:**
1. **Unusual pattern:** Most strategies degrade in OOS. Negative degradation (improvement) is rare and suggests:
   - Overfitting to OOS regime (forward-looking bias)
   - Statistical luck (small sample noise)
   - Real regime shift that favors this strategy
2. **Window 2 small sample:** 33 trades with 45% improvement could be luck
3. **Trade frequency drop:** 233→33 suggests param combination is regime-sensitive (works less frequently in 2025-2026)

**Verdict:** **NON-BLOCKING FALSE POSITIVE (with regime caveat)**

**Rationale:** The evidence leans toward real edge rather than artifact for three reasons:

1. **Window 1 is independently credible:** 106 trades with 43% OOS improvement is unlikely to be pure luck. If this were overfitting noise, we'd expect window 1 to be marginal (barely passing) rather than strongly outperforming.

2. **Architectural validation reduces artifact risk:** V3 audit confirmed that top candidates follow gate vs premium pattern. trial-00095 matches this pattern exactly (low sweep weight, high quality filter weights). This is external validation that reduces "too good to be true" concern.

3. **Regime hypothesis is plausible:** 2022-2023 was high-volatility bear market, 2024-2026 is post-halving consolidation. Quality filters (reclaim + TFI + trend) might work better in consolidation regimes (fewer false sweeps, cleaner trend structures) than in chaotic bear markets. This would explain why train (includes 2022-2023) underperforms vs val (2024-2026).

**Risk caveat:** Regime-sensitive edge. If 2026+ market conditions change (e.g., return to high-volatility chaos), this edge might weaken. Trade frequency drop (233→33) supports regime-sensitivity hypothesis.

**Monitoring condition:** Track live trade frequency and ER. If live trade frequency remains <2 trades/month OR live ER drops below 1.0 after 30 trades, re-evaluate for regime shift.
